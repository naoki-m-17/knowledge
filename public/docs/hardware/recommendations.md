# 用途別推奨モデル

性能比較に基づいた、用途別の推奨モデルは以下の通りである。

## M4 Pro を選ぶべきケース

### 並列負荷が高い作業

- **初回ビルドやクリーンビルド**: 数千ファイルを超える大規模プロジェクトの初回セットアップや、キャッシュクリア後の全ファイル再コンパイル
  - 注: 日常的な開発では差分ビルドが主流のため、この場面は稀にしか発生しない。しかし、発生した際にはM4 Proのマルチコア性能が有効となる
- **複数のDockerコンテナの常駐**: マイクロサービス開発、複数の開発環境の並行運用
- **仮想マシンの同時稼働**: 複数のVMを起動し、異なるOS環境で作業する

### リッチメディア制作

- **4K動画編集**: 複数の4Kトラックを同時にプレビューする
- **3Dモデリング**: 複雑な3Dシーンのリアルタイムレンダリング
- **アフターエフェクトの多用**: 複雑なエフェクトの同時処理

### ローカルAI推論

- **外部クラウドに頼らず、Mac内部で大規模言語モデルを高速に動かしたい**
- **Stable Diffusionでの画像生成**: ローカルで高解像度の画像を生成する
- **LLMのローカル実行**: プライバシーを重視し、クラウドを使わない

## M5 を選ぶべきケース

### レスポンス重視の作業

- **シングルコアのキレを活かした、ブラウザやエディタの軽快な操作を優先したい**
- **UIの応答性を最優先**: ウィンドウのドラッグ、メニューの表示、アニメーションの滑らかさ
- **エディタの入力レスポンス**: キーストロークの即座の反映、オートコンプリートの速度

### クラウド主体のワークフロー

- **重い処理はクラウド上のサーバーやAI（GPT-4, Cursor等）に任せ、Macは高性能な端末（シンクライアント）として使う**
- **クラウドベースの開発**: GitHub Codespaces、Gitpod、ReplitなどのクラウドIDEを利用
- **クラウドAIツール**: ChatGPT、Cursor、GitHub CopilotなどのクラウドAIツールを主に使用

### モビリティ重視

- **性能とバッテリー持ちのバランス、そして発熱の少なさを重視したい**
- **外出先での作業**: カフェ、コワーキングスペース、移動中の作業
- **長時間の会議**: バッテリーを気にせず、長時間の会議に参加できる
- **静音性**: ファンの音を気にせず、静かな環境で作業できる

## エンジニアとしての補足的視点

最後に、実務を行うエンジニアとしての視点を加えると、Next.jsやFirebaseを用いたモダンなWebアプリ開発においては、**「シングルコアの反応速度」**が開発体験の良さに直結することが多い。マルチコアを100%使い切る時間は1日の中で数パーセント（ビルド時等）に過ぎないため、もしクラウドベースのAIやリモートサーバーを主軸に置く開発スタイルであれば、あえてコア数を追わず、最新のシングルコア性能を持つM5にメモリを32GB積むという構成が、極めて合理的で「手の馴染みが良い」選択肢として浮上してくる。詳細な分析については、[エンジニア向け視点](./engineer-perspective.md)を参照。

### 推奨構成：エンジニア向け

- **チップ**: M5（無印）
- **メモリ**: 32GB以上
- **ストレージ**: 1TB以上（プロジェクトファイル、Dockerイメージ、node_modules等を考慮）
- **理由**: シングルコア性能とメモリ容量を優先し、マルチコア性能は必要最小限に

この構成により、**日常の開発体験を最大化**し、**予算を効率的に配分**できる。
