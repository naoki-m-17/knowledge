# 負荷時のマルチタスク：コア数の余力が生む差

重い処理を実行しながら、他の作業を快適に進められるかどうかは、コア数の「余力」に依存する。コア構成の詳細については、[コア構成の物理的特性](./core-architecture.md)を参照。

## M4 Pro (P-core 8つの場合) の挙動

重い計算や書き出し（メイン作業）に4〜6コア奪われても、残りのP-coreがブラウザやSNS、Web会議を処理するため、**「重い処理を待ちつつ、別の作業を快適に進める」**ことが可能。

### 典型的な使用シーン

```
[P-core 1-6] メイン作業（ビルド、エンコード、レンダリング）
[P-core 7-8] ブラウザ、Slack、Zoom、メール
[E-core 1-4] OSのバックグラウンド処理、通知
```

この構成により、重い処理中でも他の作業に影響を与えにくい。

## M5 (P-core 4つの場合) の挙動

メイン作業で4コア全てを占有された場合、並行している他作業のレスポンスに「カクつき」が生じる可能性がある。ただし、作業のプライオリティをOSが自動制御するため、フリーズには至らない。

### 典型的な使用シーン

```
[P-core 1-4] メイン作業（ビルド、エンコード、レンダリング）
[E-core 1-6] ブラウザ、Slack、Zoom、メール、OSのバックグラウンド処理
```

E-coreが6つあるため、軽量なタスクは快適に処理できるが、重い処理と同時に実行されるブラウザやエディタの操作に若干のラグが生じる可能性がある。

## 実務における影響

### M4 Proが有利な場面

- **長時間のビルド中に作業を継続**: プロダクションビルドを実行しながら、ブラウザでドキュメントを確認
- **動画エンコード中のマルチタスク**: 動画の書き出し中に、別の動画を編集したり、メールを返信
- **複数の仮想環境の同時稼働**: Dockerコンテナを複数起動しながら、開発作業を継続

### M5で十分な場面

- **軽量なビルドと開発作業**: Next.jsの開発サーバー起動中に、ブラウザで動作確認
- **クラウドベースのワークフロー**: 重い処理はクラウドに任せ、ローカルは軽量な作業のみ
- **シングルタスク重視**: 1つの作業に集中し、他のアプリは最小限に抑える

コア数の違いが電力効率に与える影響については、[電力効率とバッテリー](./power-efficiency.md)を参照。
